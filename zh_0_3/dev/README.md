# 插件机制
1. 找一个git存放公司的所有插件
2. 通过调用agent的/plugin/update接口拉取插件repo到本地
3. 在portal中配置哪些机器可以执行哪些插件
4. 插件命名方式：$step_xx.yy，需要有可执行权限，分门别类存放到repo的各个目录
5. 把采集到的数据打印到stdout
6. 如果觉得git方式不方便，可以改造agent，定期从某个http地址下载打包好的plugin.tar.gz
